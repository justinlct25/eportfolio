Module 3 - Unit 8 Artifacts

Cryptography programming exercise

Select one of the methods described/ examples given and create a python program that can take a short piece of text and encrypt it.

The Symmetric Cryptography with Data Encryption Standard (DES) algorithm was chosen to be attempted. Symmetric cryptography is the encryption and decryption process using a single key. DES is a symmetric key algorithm that uses block ciphers of 64 bits and a key size of 56 bits.

Why did you select the algorithm you chose?

Compared to the asymmetric key which works in the reverse way of symmetric cryptography and requires two keys, symmetric cryptography is simpler and faster resulting in less requirement on performance. It is also simpler for me to try implementing it compared to the asymmetric key. Due to the better performance of symmetric encryption compared to asymmetric, it is uypically used for bulk encryption or encrypting large amounts of data such as database encryption (Cryptomathic, 2020). However, it is of paramount importance that the symmetric key must be exchanged between two parties in a secure way.

Would it meet the GDPR regulations? Justify your answer.

Using symmetric cryptography with the DES algorithm can reduce the probability of data breach. Important data can be encrypted instead of being stored, passed or processed in plain text. GDPR places the responsibility on the controller and the processor of the personal data to implement appropriate measures to secure the data. Considering that GDPR does not mention the explicit encryption methods in order to secure the data, the DES algorithm for symmetric cryptography could be one of the possible ways to secure the data.

However, as Thakkar (2020) mentioned, DES has already been cracked by many security researchers. It was officially deprecated and replaced by the AES encryption algorithm in 2005. Low encryption key length is the biggest downside to DES, which enables brute-forcing attack success easily.

Besides, 3DES is an upgraded version of the DES algorithm, which was developed to overcome the drawbacks of the DES algorithm. The algorithm thrice to each data block. Nonetheless, it was still discovered with the Sweet32 vulnerability, causing the security industry to deprecate the algorithm. The draft guidance provided by the National Institute of Standards and Technology (NIST) (2019) also mentioned that the use of 3DES is to be scrapped in all new applications after 2023. TLS 1.3, which is the latest standard of SSL/TLS protocols, also discontinued using 3DES.

Therefore, the DES algorithm used may not be sufficient nowadays. More secure version of encryption may be chosen for data protection.
